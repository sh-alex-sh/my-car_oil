# 车辆油耗统计 App — 产品设计文档

> **创建日期:** 2026-06-09
> **状态:** 已确认，待进入实施计划

---

## 1. 项目概述

**一句话描述：** 个人车主使用的油耗统计 Web 应用，纯本地存储，支持数据导出，分三阶段开发。

**目标用户：** 个人车主（1-2 辆车），追求简单易用。

**核心价值：** 记录加油数据 → 自动计算油耗 → 了解车辆花费 → 数据随时导出。

---

## 2. 技术方案

| 维度 | 决策 |
|---|---|
| 平台 | Web MVP → PWA（添加到主屏幕） → 未来可迁 React Native |
| 框架 | React 18 |
| 构建工具 | Vite（秒级热更新） |
| 样式 | Tailwind CSS |
| 状态管理 | Zustand（轻量、无 boilerplate） |
| 本地数据库 | Dexie.js（IndexedDB 封装） |
| 图表 | Recharts（Phase 2） |
| 数据导出 | SheetJS (xlsx) → Excel 文件下载 |
| 路由 | React Router（Hash 路由，静态部署零配置） |
| 离线 | PWA（vite-plugin-pwa + Service Worker） |
| 部署 | 纯静态文件，任意静态托管 |

**架构图：**

```

│                    浏览器 (PWA)                   │

│  │  React   │  │ Zustand │  │   Dexie.js       │  │
```

**核心原则：**
- 零后端依赖，数据完全在用户浏览器中
- 每个页面顶部导航栏左侧带返回按钮，支持浏览器原生返回手势
- 移动端优先的响应式设计

## 4. 数据模型

### 4.1 加油记录表 `fuel_records` (Phase 1)

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | number (自增) | 主键 |
| `vehicleId` | number | 车辆 ID（Phase 1 默认=1） |
| `date` | string (ISO 8601) | 加油日期 |
| `mileage` | number | 当前里程表读数（公里） |
| `fuelAmount` | number | 加油量（升） |
| `fuelCost` | number | 加油总金额（元） |
| `fuelPrice` | number | 单价（元/升），自动计算 = fuelCost / fuelAmount |
| `isFullTank` | boolean | 是否加满 |
| `note` | string | 备注（加油站名、油品等） |
| `createdAt` | string (ISO 8601) | 创建时间 |

**油耗计算公式（加满法）：**
```
油耗 (L/100km) = 本次加油量 / (本次里程 - 上次里程) × 100
```
仅当 `isFullTank = true` 且存在前一条加满记录时才计算油耗。

### 4.2 车辆表 `vehicles` (Phase 2)

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | number (自增) | 主键 |
| `name` | string | 车辆昵称 |
| `brand` | string | 品牌 |
| `model` | string | 型号 |
| `plateNumber` | string | 车牌号 |
| `isActive` | boolean | 当前活跃车辆 |

### 4.3 后续表 (Phase 3)

- 保养提醒表 `maintenance_reminders`
- 油价记录表 `fuel_prices`
- 费用分类表 `expense_categories` / `expenses`

```
│
```

**组件层级关系：**
```
App
├── NavBar（每页面顶部，根据路由切换）
├── 路由内容区
│   ├── Dashboard → SummaryCard×4 + RecentRecords → RecordCard×5
│   ├── RecordList → FilterTabs + RecordCard×N + EmptyState
│   ├── RecordForm → DatePicker + NumberInput×2 + ConfirmDialog
│   ├── Stats（Phase 2 占位）
│   └── Settings → ConfirmDialog
└── TabBar（底部，4 个主 Tab 时显示）
```

**设计原则：**
- NavBar 和 TabBar 在 App 层统一管理
- RecordCard 在 Dashboard 和 RecordList 中复用
- 所有计算逻辑放入 `utils/calculation.ts`（纯函数，易测试）
- 组件不直接访问 Dexie.js，通过 hooks 封装数据操作

## 9. 非功能需求

- **响应式：** 适配 320px ~ 768px 屏幕宽度
- **离线：** PWA Service Worker 缓存，无网络可打开
- **性能：** 记录超 1000 条时列表虚拟滚动（Phase 2 评估）
- **隐私：** 所有数据存本地浏览器，不上传任何服务器
- **国际化：** Phase 1 仅中文，日期用 `zh-CN` 格式
